Built while just starting to learn Angular 2, I needed something that can be used as boilerplate for a single page app, something that my team can easily use right away. This can serve as boilerplate for a small to medium-sized single-page app with minimal modifications.


My problem with Angular 2 right now is that most of the resources are in TypeScript. I'm going to use Angular, My team can't use TypeScript. We have a deadline.


We're dinosaurs when it comes to javascript so the only cli command is just npm install and my JavaScript is just JavaScript. I don't have anything running on the background. Learn TypeScript when you have the time or you'll be judged accordingly. :P


If you're a designer, you can skip all the javascript files. It's supposed to take care of itself. Just familiarize yourself with the structure.


If you're a developer, this boilerplate automates some things for you. All you have to do is copy/paste a component folder and focus on building your component.

Explanation




Files here at the top are self explanatory.



This is our single-page app. It's a typical Angular 2 component. The JS, HTML, and optional CSS are in this folder together with some Angular 2 boilerplate. This is how we'll organize it.

A custom pipe. Pipes are just filters from Angular 1. This is a simple example used throughout our app to capitalize the first letter.

Let's put components in their own subfolder inside app/. Things for this component belong in here, otherwise they go in utils/ like our custom pipe.


Copy/paste this folder and change the file name (use lowercase) and use the same name for the constant (capitalize first letter) inside the script for a new component.

That's it! We'll see how it works in the next section.
