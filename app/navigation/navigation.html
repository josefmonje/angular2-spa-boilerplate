project/files/files.js

Extra route with arguments

In files.js, I added an extra route so I can have a files/ route (created and registered automatically) and a files/:file (accepts argument) because there are no optional arguments in Angular 2. In the source, I defined the one that accepts an argument before the one without an argument. This way, when angular matches routes, it's found first and we can fall back on the one that doesn't accept arguments if there's no file.

{{"app.routes[`__${name}File`] = {
  path: `${name.toLowerCase()}/:file`, 
  component: app.declarables[name], 
  hidden: true,
}"}}

project/app/modules.js

Private and hidden routes

Simliar to the extra route in files.js, I added a "private" __otherwise route.


__otherwise is prefixed with '__' (double underscore), which excludes it from being returned by get_list().


_FilesFile has a property 'hidden: true' that excludes it from being displayed in the nav.

{{"app.routes.__otherwise = { path: '**',
  redirectTo: 'home',
  pathMatch: 'full',
}"}}
